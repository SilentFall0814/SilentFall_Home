# 如何用服务器的宝塔面板将博客项目部署到你的服务器中

> 适用对象：将当前 `Vite + React` 静态前端项目部署到安装了宝塔面板的 Linux 服务器中。本文以本仓库当前产物 `dist` 目录为主线，同时补充未来扩展到 Node 或数据库场景时的建议配置。

## 一、前置准备

### 1. 服务器基础要求

建议最低配置如下：

- CPU：`2` 核
- 内存：`2 GB`
- 系统盘：`40 GB SSD`
- 带宽：`3 Mbps` 及以上
- 操作系统：`Ubuntu 22.04 LTS`、`Debian 12`、`CentOS Stream 9` 任选其一

如果你的站点后续还会接入 Node 接口服务、数据库或后台管理系统，建议直接使用：

- CPU：`2` 到 `4` 核
- 内存：`4 GB` 及以上
- 带宽：`5 Mbps` 及以上

### 2. 域名与证书准备

在部署前，建议提前准备好以下内容：

- 一个已经注册完成的域名
- 域名管理后台权限
- 服务器公网 IP
- 如果所在地区有要求，提前完成备案
- 如果你打算直接启用 HTTPS，建议准备：
  - `Let's Encrypt` 免费证书
  - 或你自己申请好的商业证书

### 3. 博客项目打包输出规范

当前仓库是静态前端项目，正式部署时应以构建产物 `dist` 目录为准。

本地执行：

```bash
npm install
npm run build
```

构建成功后，会生成：

```text
dist/
├─ index.html
├─ assets/
└─ 其他静态资源
```

部署时上传的应该是 `dist` 目录内的内容，而不是源码目录。

### 4. 宝塔面板安装步骤

如果服务器尚未安装宝塔面板，可以在服务器中使用官方安装脚本。

常见 Linux 环境示例：

```bash
if [ -f /usr/bin/curl ];then curl -sSO https://download.bt.cn/install/install_panel.sh;else wget -O install_panel.sh https://download.bt.cn/install/install_panel.sh;fi && bash install_panel.sh ed8484bec
```

安装完成后，终端会输出：

- 面板访问地址
- 初始用户名
- 初始密码

请立即记录并妥善保管。

### 5. 宝塔初始化配置

首次登录宝塔后，建议先完成以下操作：

1. 修改面板登录用户名和密码
2. 绑定宝塔账号
3. 设置面板安全入口
4. 限制登录 IP 或启用堡垒机访问策略
5. 更新面板到稳定版本

## 二、宝塔面板环境配置

### 1. 当前项目推荐环境

因为当前项目是纯静态前端站点，所以推荐环境非常简单：

- Web 服务器：`Nginx`
- 运行方式：静态站点
- 数据库：当前不需要
- Node.js：当前不需要常驻运行

也就是说，你不需要为了当前项目额外启动 Node 进程守护，只需要把 `dist` 放到站点目录并正确配置 Nginx 即可。

### 2. 安装 Nginx

在宝塔后台进入：

```text
软件商店 -> 运行环境 -> Nginx
```

点击安装，等待安装完成。

截图建议：

- 这里建议截图“软件商店 -> Nginx 安装页”

### 3. 创建站点

进入：

```text
网站 -> 添加站点
```

建议填写方式如下：

- 域名：`你的域名`，例如 `example.com`
- 根目录：`/www/wwwroot/你的域名`
- PHP 版本：选择“纯静态”
- 数据库：不创建

创建完成后，宝塔会自动生成一个 Nginx 站点配置。

截图建议：

- 这里建议截图“添加站点”表单页

### 4. 如果未来需要 Node / PHP / 数据库

虽然当前项目不需要，但如果你后续博客改造成动态项目，可以按下面思路准备：

- Node.js 项目：
  - 安装 `Node.js` 管理器
  - 使用宝塔 `PM2` 或进程守护启动服务
  - 通过 `Nginx` 反向代理到 Node 端口
- PHP 博客：
  - 安装 `PHP`、`MySQL` 或 `MariaDB`
  - 为站点绑定 PHP 版本
- 数据库项目：
  - 在 `数据库` 中创建专用账户
  - 给该账户最小必要权限

这些属于扩展方案，不是当前静态站点部署的必需项。

## 三、项目上传与部署

### 1. 方式一：宝塔面板直接上传

这是最适合新手的方式。

步骤如下：

1. 本地执行：

```bash
npm run build
```

2. 打开宝塔：

```text
文件 -> /www/wwwroot/你的域名
```

3. 删除站点目录中默认生成的无用文件
4. 上传本地 `dist` 目录中的全部文件
5. 确保最终结构类似：

```text
/www/wwwroot/你的域名
├─ index.html
├─ assets/
└─ 其他静态资源
```

截图建议：

- 这里建议截图“文件管理器中的站点根目录”

### 2. 方式二：Git 拉取部署

如果你的项目放在 GitHub、Gitee 或私有 Git 平台，可以直接在服务器拉取代码。

步骤如下：

1. 在服务器安装 Git
2. 进入站点目录或临时目录
3. 拉取项目：

```bash
git clone 你的仓库地址
cd 项目目录
npm install
npm run build
```

4. 将 `dist` 目录内容复制到站点根目录

这种方式适合后续持续更新，但对服务器 Node 环境和 Git 权限有要求。

### 3. 方式三：SFTP 传输

如果你习惯用 `Xftp`、`FileZilla` 或 `WinSCP`，也可以通过 SFTP 上传构建后的产物。

基本步骤：

1. 本地执行 `npm run build`
2. 使用 SFTP 连接服务器
3. 上传 `dist` 目录中的文件到：

```text
/www/wwwroot/你的域名
```

### 4. 静态资源托管路径说明

当前项目构建后默认使用以根路径 `/` 开头的资源引用，因此：

- 如果你将站点部署到独立域名根目录，例如 `https://example.com/`，无需额外修改
- 如果你要部署到子路径，例如 `https://example.com/home/`，则需要修改 `Vite` 的 `base` 配置后重新构建

### 5. 当前项目是否需要运行端口

当前项目属于静态站点：

- 不需要暴露单独的 Node 运行端口
- 不需要反向代理
- 不需要进程守护

只有在未来改造成 Node SSR、接口服务或前后端一体项目时，才需要：

- 配置端口
- 配置反向代理
- 配置进程守护

### 6. 环境变量配置说明

当前纯静态站点没有运行时环境变量依赖。

如果未来改造为带接口地址的前端项目，推荐：

- 在构建前通过 `.env.production` 注入变量
- 避免把敏感密钥写进前端构建产物

## 四、域名与安全配置

### 1. 域名解析

在域名服务商后台添加一条 `A` 记录：

- 主机记录：`@`
- 记录类型：`A`
- 记录值：你的服务器公网 IP

如果你还需要 `www` 域名：

- 主机记录：`www`
- 记录类型：`A`
- 记录值：同一个服务器公网 IP

### 2. 申请 Let’s Encrypt 免费证书

进入：

```text
网站 -> 你的站点 -> 设置 -> SSL
```

选择：

- `Let's Encrypt`

然后：

1. 勾选域名
2. 点击申请
3. 申请成功后启用证书

截图建议：

- 这里建议截图“站点设置 -> SSL”页签

### 3. 上传自定义证书

如果你已经有自己的证书文件，也可以直接上传：

- 证书文件：`.pem` 或 `.crt`
- 私钥文件：`.key`

仍然进入：

```text
网站 -> 你的站点 -> 设置 -> SSL
```

选择“其他证书”后粘贴证书内容即可。

### 4. 开启 HTTPS 强制跳转

证书启用后，打开：

```text
强制 HTTPS
```

这样用户无论访问 `http://` 还是 `https://`，都会被统一跳转到加密连接。

### 5. 配置防火墙规则

当前静态站点至少放行以下端口：

- `22`：SSH
- `80`：HTTP
- `443`：HTTPS

如果你还开启了宝塔面板，请确保面板端口也可访问，但最好限制访问来源。

### 6. 跨域与安全头建议

当前纯静态主页通常不需要复杂跨域配置。

如果你后续接入外部 API，可以在 Nginx 中按需补充：

- `Access-Control-Allow-Origin`
- `X-Frame-Options`
- `X-Content-Type-Options`
- `Referrer-Policy`

对于纯静态站点，推荐至少保留以下安全思路：

- 禁止随意目录浏览
- 仅开放必要端口
- 面板入口不要暴露默认路径

## 五、项目启动与测试

### 1. 当前静态站点的启动方式

当前项目上传完成后，不需要额外“启动”。

只要：

- Nginx 已安装
- 站点已创建
- `dist` 内容已上传到根目录

你就可以直接通过域名访问。

### 2. 本地 hosts 调试

如果域名还没正式切换，或者你想先在自己的电脑验证，可以修改本地 `hosts`：

Windows 路径：

```text
C:\Windows\System32\drivers\etc\hosts
```

添加：

```text
你的服务器IP    你的域名
```

保存后，再在浏览器中访问该域名进行预检。

### 3. 正式访问测试清单

上线后建议逐项检查：

1. 域名是否可访问
2. `http` 是否会自动跳转到 `https`
3. 首页资源是否全部加载成功
4. 锚点导航是否可用
5. 外链是否正常打开
6. 手机端布局是否正常

### 4. 常见故障排查

#### 502 / 503 错误

当前静态站点一般不会出现这类错误。

如果出现，多半是：

- Nginx 配置错误
- 你把站点错误地配置成了反向代理模式
- 后续扩展到 Node 服务时，Node 进程未启动

#### 静态资源 404

优先检查：

- 上传的是否是 `dist` 目录内容，而不是源码目录
- `assets` 目录是否完整
- 站点根目录是否正确
- 是否部署到了子路径但没改 `base`

#### 数据库连接失败

当前项目没有数据库，不会出现该问题。

如果你未来做成动态博客，再检查：

- 数据库地址
- 账户权限
- 防火墙端口
- 环境变量

## 六、日常维护

### 1. 日志查看

进入：

```text
网站 -> 你的站点 -> 日志
```

重点查看：

- 访问日志
- 错误日志

如果页面资源加载异常，优先查看错误日志。

### 2. 数据备份

虽然当前项目是静态站点，也建议做好备份。

至少备份：

- 网站根目录文件
- 如果未来接了数据库，再一起备份数据库

宝塔中可用：

```text
计划任务 -> 备份网站
计划任务 -> 备份数据库
```

### 3. 版本更新部署

推荐流程：

1. 本地修改并测试
2. 本地执行：

```bash
npm run build
```

3. 备份服务器当前站点目录
4. 上传新的 `dist` 内容覆盖旧版本
5. 刷新站点并核验页面

### 4. 回滚方案

建议每次更新前保留一个时间戳备份目录，例如：

```text
/www/backup/站点名-2026-06-11
```

如果新版本出现问题，可以快速把旧版本恢复回来。

### 5. 资源监控

在宝塔首页关注以下指标：

- CPU
- 内存
- 磁盘占用
- 带宽

虽然当前静态站点资源占用通常不高，但如果未来加了 Node 服务或数据库，监控会非常重要。

## 八、当前项目的推荐部署结论

对于当前这个仓库，最推荐的方案是：

1. 本地执行 `npm run build`
2. 将 `dist` 目录内容上传到宝塔站点根目录
3. 使用 `Nginx + 静态站点` 方式部署
4. 配置域名解析和 HTTPS
5. 完成访问验证与日常备份

这也是当前仓库最稳、最简单、最容易长期维护的上线方式。
